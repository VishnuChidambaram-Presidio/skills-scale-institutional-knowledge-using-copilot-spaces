# OctoAcme Role Onboarding Checklist

## Purpose
This checklist helps new team members quickly onboard to their role on OctoAcme projects. Complete the relevant sections based on your role to ensure you have access, context, and connections needed to be effective.

---

## General Onboarding (All Roles)

### Week 1: Access & Context
- [ ] Receive project overview and charter document
- [ ] Get added to relevant Slack/Teams channels
- [ ] Gain access to project repository and documentation
- [ ] Access project board (GitHub Projects, Jira, etc.)
- [ ] Review OctoAcme project management documentation:
  - [ ] [Project Management Overview](./octoacme-project-management-overview.md)
  - [ ] [Roles and Personas](./octoacme-roles-and-personas.md)
  - [ ] [Role Interaction Matrix](./octoacme-role-interaction-matrix.md)
- [ ] Meet Project Manager for role expectations discussion
- [ ] Meet key collaborators (see role-specific sections below)
- [ ] Receive calendar invites for recurring meetings

### Week 2: Integration
- [ ] Attend first standup/team meeting
- [ ] Shadow key role-specific activities (see below)
- [ ] Review current sprint/iteration goals
- [ ] Understand current project status and critical path
- [ ] Identify any blockers or missing access
- [ ] Start contributing to assigned tasks

### Ongoing
- [ ] Complete required training (security, compliance, tools)
- [ ] Provide onboarding feedback to Project Manager
- [ ] Update this checklist with lessons learned

## Role-Specific Onboarding

### Project Manager

**Key Collaborators to Meet:**
- [ ] Product Manager - product vision and roadmap alignment
- [ ] Product Owner - backlog priorities and acceptance criteria
- [ ] Development Lead - technical approach and team structure
- [ ] QA Lead - quality strategy and test approach
- [ ] DevOps Engineer - deployment and infrastructure overview
- [ ] Change Manager - change control process
- [ ] Key Stakeholders - expectations and communication preferences

**Access & Tools:**
- [ ] Project management tool (edit access)
- [ ] Risk register and issue log
- [ ] Budget and resource tracking tools
- [ ] Status report templates
- [ ] Stakeholder distribution lists

**Key Activities:**
- [ ] Shadow previous PM in status meeting
- [ ] Review risk register and mitigation plans
- [ ] Review project timeline and dependencies
- [ ] Understand escalation paths
- [ ] Review communication plan and cadence

---

## Onboarding Completion Checklist

### After 30 Days
- [ ] Can independently perform core role responsibilities
- [ ] Understand team norms and collaboration patterns
- [ ] Have established working relationships with key collaborators
- [ ] Contributing value to project goals
- [ ] No major blockers or missing access

### Feedback & Improvement
- [ ] Complete onboarding survey (if available)
- [ ] Provide feedback to Project Manager on onboarding experience
- [ ] Suggest improvements to this checklist
- [ ] Help onboard the next team member

---

## Tips for Success

1. **Ask Questions Early**: Don't wait until you're stuck - ask for clarification
2. **Document as You Learn**: Capture answers to questions that might help others
3. **Build Relationships**: Strong collaboration starts with knowing your teammates
4. **Find a Buddy**: Identify a peer who can answer quick questions
5. **Review Documentation**: Read existing docs before asking questions
6. **Provide Feedback**: Help improve the onboarding process for future team members
